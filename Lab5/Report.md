# Отчет по лабораторной работе 5: Контроль доступа к веб-приложению

## Цель
Реализовать обязательную аутентификацию и ролевой контроль доступа (RBAC).

## Базовая уязвимость (из предыдущих работ)
До внедрения контроля сессий прямое открытие `admin.php` по URL давало доступ без входа.
В этой лабораторной работе проблема устранена.

## Реализовано
- Добавлена безопасная обработка сессий в `auth.php`:
  - запуск сессии,
  - security-флаги cookie сессии,
  - регенерация session ID после входа,
  - выход с очисткой сессии.
- Добавлено поле роли в таблицу `user` со значениями:
  - `administrator`,
  - `manager`.
- Добавлены таблицы RBAC:
  - `role`,
  - `permission`,
  - `role_permission`.
- Добавлен ролевой редирект после входа:
  - administrator -> `admin.php`,
  - manager -> `manager.php`.
- Добавлены защищенные функции менеджера:
  - просмотр учетных записей (`accounts.php`),
  - редактирование/удаление учетных записей (`edit_user.php`, `delete_user.php`).
- Добавлены обязательные проверки прав:
  - `require_role()` и `require_permission()`.
- Добавлен сценарий выхода из обеих закрытых зон через `logout.php`.

## Проверка контроля доступа
1. Неаутентифицированный пользователь при открытии `admin.php` или `manager.php` перенаправляется на страницу входа.
2. Учетная запись `administrator` может входить только в панель администратора.
3. Учетная запись `manager` может входить в панель менеджера и использовать функции управления учетными записями.
4. Ограниченные действия блокируются при отсутствии требуемого разрешения.
5. Очистка сессии при выходе предотвращает доступ через кнопку "назад"/повторное открытие URL.

## Как запустить
1. Инициализируйте БД:
   - `mysql -u root -p < init.sql`
2. Запустите сервер:
   - `php -S localhost:8000`
3. Тестовые учетные записи:
   - `admin / admin123`
   - `manager / manager123`

## Вывод
Лабораторная работа 5 обеспечивает аутентификацию и ролевые разрешения, чтобы защищенные ресурсы нельзя было открыть подбором URL или при отсутствии проверок сессии.
